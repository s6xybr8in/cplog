# CP-Log

Competitive Programming 개인 노트 & 블로그 퍼블리셔. 백엔드 없이 브라우저에서만 동작하는 Vite + React SPA입니다.

- **To Solve 보드** — 링크/텍스트 붙여넣기로 즉시 문제 등록(플랫폼·이름 자동 파싱), 고밀도 리스트 + 칸반 토글, 복습 큐
- **에디터** — CodeMirror 6, KaTeX 수식, 코드 하이라이팅, 스니펫, 옵시디언식 `[[위키링크]]` + 백링크
- **저지 연동** — Codeforces/AtCoder 핸들만 넣으면 AC 자동 반영, CF 레이팅 자동 채우기
- **GitHub 동기화** — 노트/문제를 개인 private 리포(`cplog-data`)에 마크다운으로 동기화 (Obsidian 호환)
- **원클릭 발행** — Front Matter를 붙여 GitHub Pages 블로그(`_posts/`)로 커밋
- **통계** — 활동 히트맵, 태그별 해결 분포

모든 데이터(PAT 포함)는 브라우저 localStorage와 본인 GitHub 리포에만 저장됩니다.

## 사용법

### 처음 설정

사이드바의 설정(⚙)을 열고 다음을 입력합니다. GitHub 기능을 쓰지 않아도 보드/에디터는 그대로 동작합니다.

1. **GitHub 블로그 설정** — Username, 블로그 리포 이름(예: `octocat.github.io`), 브랜치, PAT. PAT는 대상 리포에 대한 **Contents 읽기/쓰기** 권한(fine-grained) 또는 `repo` 스코프(classic)가 필요하며, 이 브라우저의 localStorage에만 저장됩니다.
2. **데이터 저장소 동기화** — 노트/문제를 백업할 리포 이름(기본 `cplog-data`)을 정하고 **[데이터 리포 생성 (private)]** 버튼으로 원클릭 생성. 자동 동기화를 켜면 시작 시 가져오고 편집 후 몇 초 뒤 자동 업로드합니다.
3. **저지 연동** — Codeforces / AtCoder 핸들 입력 (공개 API라 토큰 불필요).

### 문제 관리 (To Solve 보드)

- 상단 캡처 바에 **문제 URL을 붙여넣으면 즉시 등록**됩니다 — Codeforces / AtCoder / BOJ / USACO 링크는 플랫폼과 문제 이름을 자동 인식합니다. 여러 줄을 한 번에 붙여넣으면 줄 단위로 일괄 등록되고, 일반 텍스트는 Enter로 추가합니다.
- 기본은 고밀도 **리스트 뷰**(행 클릭으로 수정, 상태는 셀렉트로 변경), 헤더 토글로 **칸반 뷰**(드래그 앤 드롭)로 전환할 수 있습니다.
- 문제를 **Done**으로 바꾸면 복습 시점(3일 뒤 / 1주 뒤 / 1달 뒤 / 안 함)을 물은 뒤 해당 문제의 **오답 노트가 자동 생성**되어 에디터로 이동합니다.
- 보드 상단 **복습 스트립**에 복습 예정 문제가 기한순으로 표시됩니다. 기한이 지난 칩은 강조되며, 칩에서 바로 "다시 풀기"(In Progress로 복귀) 또는 "복습 완료"를 선택합니다.

### 에디터

- **Ctrl+E**로 편집 ↔ 미리보기를 전환합니다. 미리보기는 KaTeX 수식(`$O(N \log N)$`)과 C++/Python 등 코드 블록 하이라이팅을 지원합니다.
- **스니펫**: 메타바의 스니펫 드롭다운에서 검색해 커서 위치에 삽입합니다. `{{date}}`, `{{time}}`, `{{datetime}}`, `{{title}}`, `{{cursor}}` 변수가 치환되며, 스니펫 관리 모달에서 생성/편집/`.md` 가져오기·내보내기가 가능합니다.
- **위키링크**: `[[`를 입력하면 노트 제목·문제 이름 자동완성이 뜹니다 (**Ctrl+Shift+L**로도 호출). 미리보기에서 `[[링크]]`를 클릭하면 해당 노트로 이동하고, 없는 노트면 그 제목으로 새로 만듭니다. 에디터 하단 **백링크 바**에 현재 노트를 언급한 노트들이 표시됩니다.
- **세션 노트**: Ctrl+K → "오늘 세션 노트 만들기" — 셋 이름과 문제 수를 고르면 `YYMMDD 셋이름` 제목에 `## A`~ 헤딩과 `# Upsolving` 섹션이 채워진 데일리 노트가 생성됩니다.

### 블로그 발행

에디터 우측 상단 **Publish** 버튼을 누르면 title/date/categories/tags Front Matter를 붙여 블로그 리포의 `_posts/YYYY-MM-DD-제목.md`로 커밋합니다. 카테고리·태그는 에디터 메타바에서 편집하고, `[[위키링크]]`는 발행 시 일반 텍스트로 변환됩니다. 같은 날 다시 발행하면 기존 파일이 갱신됩니다 (파일명의 날짜는 발행 시점 기준).

### 동기화

데이터 리포에는 보드가 `problems.json`, 노트가 `notes/*.md`, 스니펫이 `snippets/*.md`로 저장됩니다 — 리포를 클론하면 Obsidian 등에서 바로 열 수 있습니다. 사이드바 하단 인디케이터로 상태(동기화됨 / 대기 중 / 오류)를 확인하고, 클릭하거나 **Ctrl+S**로 즉시 동기화합니다. PAT·설정·테마는 리포에 올라가지 않습니다.

### 저지 연동

핸들을 등록하면 앱 시작 시(또는 Ctrl+K → "저지 동기화") 제출 이력을 읽어 **AC된 보드 문제를 자동으로 Done 처리**하고 해결 시각을 기록합니다. 난이도가 비어 있는 Codeforces 문제는 레이팅을 자동으로 채웁니다. BOJ는 solved.ac가 브라우저 호출을 차단해 지원하지 않습니다.

### 통계

사이드바 "통계"에서 스탯 타일, 최근 26주 활동 히트맵(문제 해결 + 노트 작성), 태그별 해결 분포를 봅니다.

### 단축키

| 키 | 동작 |
| --- | --- |
| `Ctrl+K` | 퀵 스위처 — 노트/문제/명령 통합 검색 (노트는 본문까지 검색) |
| `Ctrl+E` | 편집 ↔ 미리보기 전환 (보드에서 누르면 에디터로 이동) |
| `Ctrl+S` | 지금 동기화 (미설정 시 설정 모달) |
| `Ctrl+Alt+N` | 새 노트 |
| `Ctrl+Shift+L` | 위키링크 자동완성 삽입 |
| `Ctrl+F` | 에디터 내 검색 |
| `Ctrl+Shift+[` / `]` | 커서 위치의 섹션·코드블록 접기 / 펼치기 |
| `Ctrl+Alt+[` / `]` | 전체 접기 / 전체 펼치기 |

## Windows 앱

브라우저 없이 데스크톱 앱으로도 쓸 수 있습니다. 웹 버전과 같은 화면이며, 데이터는 앱 자체 저장소에 저장되므로 브라우저에서 쓰던 노트는 **데이터 리포 동기화로 가져옵니다**(설정 → 데이터 저장소 동기화).

`release/` 폴더의 두 가지 중 하나를 쓰면 됩니다.

- `CP-Log Setup 0.1.0.exe` — 설치 파일 (시작 메뉴·바탕화면 바로가기 생성, 설치 위치 선택 가능)
- `CP-Log-0.1.0-portable.exe` — 설치 없이 실행하는 단일 실행 파일

> 코드 서명을 하지 않아서 첫 실행 때 Windows SmartScreen 경고가 뜹니다. **추가 정보 → 실행**으로 진행하세요.

직접 빌드하려면:

```bash
npm run dist:win     # release/ 에 설치 파일 + 포터블 exe 생성
npm run electron:dev # 개발용 (먼저 다른 터미널에서 npm run dev)
```

`npm install` 후 `node_modules/electron/dist/`가 비어 있으면 `node node_modules/electron/install.js`로 Electron 바이너리를 내려받습니다.

## 개발

```bash
npm install
npm run dev
```

`main` 브랜치에 push하면 GitHub Actions가 자동으로 GitHub Pages에 배포합니다.
