# 프로젝트 명: CP-Log (Competitive Programming 개인 노시디언 & 블로그 퍼블리셔)

## 📌 역할 설정
너는 프론트엔드 개발 및 생산성 도구 설계의 전문가야. React, Tailwind CSS, Lucide-react 아이콘을 활용하여 동작하는 인터랙티브 웹 애플리케이션(Artifacts)을 만들어줘. UI는 깔끔하고 직관적인 다크(Dark) 테마 기반의 모던한 디자인으로 구성해 줘.

## 🚀 핵심 목표
알고리즘 문제 해결(CP/PS) 학습을 위한 개인용 노션(Notion) 스타일의 에디터이자, 작성한 오답 노트와 풀이를 GitHub Pages(기술 블로그)로 원클릭 자동 푸시하는 통합 웹앱을 구축한다. (주요 타겟 플랫폼: Codeforces, AtCoder, USACO, KOI)

## ⚠️ 반드시 지켜야 할 기술 제약 (검증으로 확인된 사실)
- **`X-GitHub-Api-Version` 헤더를 브라우저 fetch에 절대 추가하지 말 것** — GitHub CORS preflight 허용 목록에 없어 요청 자체가 실패함 (github/docs#24706). `Authorization: Bearer` + `Accept: application/vnd.github+json`만 사용 (`ghFetch` 헬퍼 경유).
- UTF-8→base64는 `TextEncoder`/`TextDecoder` 기반 (`toBase64`/`fromBase64`). `unescape()` 금지. **바이너리(이미지)는 다름** — `blobToBase64`(FileReader readAsDataURL)로 인코딩, `base64ToBytes`(atob→Uint8Array)로 디코딩. `fromBase64`(UTF-8 문자열)를 이미지에 쓰면 깨짐.
- **이미지 에셋은 private 데이터 리포에 있어 raw URL(`raw.githubusercontent.com`)로 브라우저 `<img>`에 직접 표시 불가**(토큰 헤더 못 붙임) — 반드시 IndexedDB 캐시 또는 Contents API(`GET /contents/assets/…`, base64 응답) 경유해 blob→objectURL로 표시(`resolveAsset`/`AssetImage`). 기능 #18 참조.
- 동기화 엔진의 path→sha 맵(`filesRef`)은 **in-memory 단일 진실 공급원** — pull/push가 직접 갱신하고 localStorage로 미러링만 함. React 상태 왕복에 의존하면 pull 직후 push가 stale 맵을 읽는 버그 재발 (실제로 한 번 발생해 수정함).
- PAT·GitHub 설정·테마는 절대 데이터 리포에 푸시하지 않음.
- 검증 방법: Playwright + `page.route('https://api.github.com/**')` 목킹으로 E2E (트리/컨텐츠 GET, PUT 바디의 sha·front matter 검증). 임시 검증 스크립트는 검증 후 삭제.
- **저지 API CORS (2026-07-13 브라우저 실측)**: codeforces.com/api(user.status, problemset.problems)와 kenkoooo.com/atcoder/atcoder-api/v3(user/submissions)는 브라우저 fetch 허용. **solved.ac API와 kenkoooo problems.json(S3 정적)은 차단** — BOJ 티어 조회·AtCoder 난이도는 프록시 없이 불가하니 재시도하지 말 것.
- **vite.config.js의 `server.watch.ignored: ['**/.omc/**']`를 제거하지 말 것** — OMC(에이전트 툴링) 훅이 `.omc/state/*.json`을 수 초 간격으로 갱신해 dev 서버가 풀 리로드를 반복, E2E가 랜덤한 지점에서 실패하는 가짜 플레이크를 만든다 (Ctrl+K E2E에서 실제 발생해 수정).

Tauri는 이 PC에 Rust 툴체인이 없어 제외(VS2022 빌드 도구는 있음). 렌더러는 **웹 버전과 완전히 같은 번들** — 앱 코드에 Electron 전용 분기가 하나도 없다. 셸은 `electron/main.cjs` 한 파일(`"type": "module"`이라 확장자는 `.cjs`), 설정은 `electron-builder.yml`, 아이콘은 `build/icon.ico`(사이드바 로고를 Playwright로 렌더→PNG 캡처→ICO 컨테이너로 감쌈).
- 스크립트: `npm run build:app`(=`vite build --base=./ --outDir dist-electron`, 상대 base라야 app:// 아래에서 에셋이 풀림) / `npm run electron:dev`(별도 터미널에서 `npm run dev` 먼저 — `--dev` 플래그로 5173 로드, Windows npm 스크립트의 env 주입이 셸마다 달라 플래그 방식) / `npm run dist:win`(설치 파일 + 포터블 exe).
- **★`file://`로 띄우면 안 된다** — 오리진이 불투명(opaque)해 localStorage/IndexedDB가 보장되지 않아 노트가 저장되지 않는다. `protocol.registerSchemesAsPrivileged`로 `app://`을 standard·secure·corsEnabled로 등록하고 `protocol.handle`에서 `net.fetch(pathToFileURL(...))`로 dist-electron을 서빙한다(경로 탈출 차단 포함). 검증: `origin=app://cplog`, `isSecureContext=true`, localStorage·IndexedDB 동작, 재시작 후 노트 유지.
- 외부 링크(Codeforces 등)는 `setWindowOpenHandler` + `will-navigate`에서 `shell.openExternal`로 보내 앱 창을 덮어쓰지 않게 한다. 렌더러는 `contextIsolation: true` / `nodeIntegration: false`(preload 없음 — IPC를 쓰지 않는다).
- **함정 1 — Electron 바이너리가 자동으로 안 받아진다**: npm 11의 allow-scripts 게이팅으로 `npm install` 후 `node_modules/electron/dist/`가 비어 있을 수 있다. 그땐 `node node_modules/electron/install.js`를 직접 실행할 것. (루트 `postinstall`로 넣으면 GitHub Pages CI의 `npm ci`가 100MB를 헛으로 받으므로 넣지 않았다.)
- **함정 2 — 프로젝트 폴더 안에서 `electron-builder`가 EPERM으로 실패한다**: `release/win-unpacked.tmp` → `win-unpacked` 이름 변경이 막힌다. 원인은 **에디터(VS Code)의 재귀 파일 워처**가 새로 생긴 디렉터리에 핸들을 잡는 것 — 60초 대기해도 안 풀리고, 샌드박스·OneDrive·Defender·exe 여부는 모두 배제됐으며(같은 내용을 Temp에 두면 정상, 프로젝트 안에 200MB 디렉터리를 새로 복사만 해도 rename이 즉시 EPERM), `.vscode/settings.json`의 `files.watcherExclude`에 `release`/`dist-electron`을 넣어 해결한다(**창 다시 로드 필요**). 급하면 출력 경로를 프로젝트 밖으로: `npx electron-builder --win --config.directories.output=$env:TEMP/cplog-release`.
- 산출물은 `release/`(gitignore): `CP-Log Setup 0.1.0.exe`(NSIS, 설치 위치 선택 가능·사용자 단위 설치) + `CP-Log-0.1.0-portable.exe`. **코드 서명 없음** → 첫 실행 시 SmartScreen 경고가 뜬다(개인용이라 그대로 둠).
- 검증: Playwright `_electron`으로 (a) 개발 트리 실행 13건 — app:// 로드·localStorage/IndexedDB·재시작 후 노트 유지·CodeMirror/폴딩·외부 링크가 새 창을 열지 않음·pageerror 0, (b) **패키징된 `CP-Log.exe`** 실행 5건 — asar 안에서도 app://·localStorage·에디터 진입 정상.
